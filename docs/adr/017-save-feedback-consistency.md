# ADR-017: 保存フィードバックの統一

## Status: Accepted

## Context
モデルタブの「保存」ボタンを押しても「保存しました」が表示されない。
他のタブ（API Key, MCP）では表示される。ユーザビリティの統一が取れていない。

## Decision
全タブの保存操作で `t('settings.saved')` を表示する。

### チェック対象
- [x] API Key タブ — 保存フィードバックあり
- [ ] モデルタブ — **なし → 修正**
- [x] MCP タブ — 保存フィードバックあり
- [ ] テンプレートタブ — **なし → 修正**
- [ ] スキルタブ — **なし → 修正**

### ルール
保存操作後は必ず以下を実行:
1. `setStatus('saved')` または同等の state 更新
2. 2秒後に `setStatus('idle')` でフィードバックを消す
3. `{t('settings.saved')}` を表示

## Consequences
- 全タブで統一された保存フィードバック
- ユーザーが「保存されたか？」を迷わない
