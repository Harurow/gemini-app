# ADR-008: i18n リグレッション防止

## Status: Accepted

## Context
コンポーネントを修正するたびに `useI18n` の `t()` 呼び出しが消え、ハードコードされた英語文字列に戻るリグレッションが繰り返し発生。

## Decision
以下のルールを厳守する:

1. **UIに表示される全文字列は `t()` を経由する** — 例外なし
2. **コンポーネントを書き直す場合、既存の `t()` 呼び出しを維持する**
3. **新しい文字列を追加する場合、`locales.ts` の ja/en 両方にキーを追加する**
4. **コンポーネント修正後、日本語表示を目視確認する**

## Consequences
- 全コンポーネント修正時に i18n チェックが必須
- `locales.ts` のキーとコンポーネントの `t()` 呼び出しの整合性を常に維持
