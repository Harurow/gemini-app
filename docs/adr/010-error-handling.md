# ADR-010: エラーハンドリングポリシー

## Status: Accepted

## Context
ユーザーからの指摘: エラー発生時のロジックが不十分。認証エラー、ネットワークエラー、API エラー等が適切にユーザーに伝わらない。

## Decision
以下のエラーハンドリングポリシーを適用する:

### 1. エラーの分類と表示
| エラー種別 | 表示方法 | ユーザーへのガイド |
|-----------|---------|-----------------|
| 認証エラー (401/403) | チャットエリアにエラーバナー | 「設定で API Key を確認してください」 |
| レート制限 (429) | エラーバナー + 自動リトライ | 「しばらく待ってから再試行してください」 |
| ネットワークエラー | エラーバナー | 「インターネット接続を確認してください」 |
| モデルエラー (400) | エラーバナー | エラーメッセージをそのまま表示 |
| MCP サーバーエラー | ツールコールブロック内に表示 | サーバー名とエラー詳細 |
| 内部エラー | エラーバナー | 「予期しないエラーが発生しました」 |

### 2. エラー処理の原則
- **throw しない**: IPC ハンドラでは throw せず、stream-error イベントで通知
- **ユーザーに必ず伝える**: サイレントに失敗しない
- **回復方法を提示**: 何をすればよいかをメッセージに含める
- **ストリーミング状態をリセット**: エラー時に isStreaming を false に戻す

### 3. ログ
- Main Process で `console.error` でログ
- Renderer のエラーバナーにはユーザー向けメッセージのみ

## Consequences
- 全エラーパスでユーザーにフィードバックが返る
- エラー状態からの復帰が可能
