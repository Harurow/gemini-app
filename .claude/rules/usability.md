## ユーザビリティ原則

- **フィードバック必須**: ユーザーの操作に対して必ず視覚的フィードバック
  - 送信 → 入力欄クリア + ストリーミング開始表示
  - エラー → エラーバナー表示 + 回復方法の提示
  - 設定保存 → 「保存しました」表示
  - ツール実行 → ステータスアニメーション

- **破壊的操作の保護**:
  - 設定モーダル: オーバーレイクリックで閉じない (X ボタンのみ)
  - セッション削除: 確認不要 (復元機能を将来追加)
  - ツール実行: shell/write は確認ダイアログ

- **状態の可視化**:
  - 処理中: Spinner + ステータステキスト
  - 接続状態: MCP サーバーの緑/赤インジケータ
  - 認証状態: API Key の有効/無効表示
