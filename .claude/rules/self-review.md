## 変更後の自己レビュー (必須)

1. **ビルド確認**: `npm start` でエラーなく起動するか
2. **i18n 確認**: ハードコード英語がないか (ADR-008, i18n-check.md)
3. **ADR 整合性**: 新しい技術選択や設計変更があれば ADR を追加/更新したか
4. **エラーハンドリング**: ユーザーにエラーが伝わるか (ADR-010)
5. **既存機能の破壊なし**: 変更が他の機能に影響していないか

## コンポーネント修正ルール

- **Edit (部分変更) を優先**。Write (全置換) は最終手段
- 全置換する場合: 修正前に Read で全体を把握し、既存の t() / import を維持
- 修正後に `grep '>A-Z'` でハードコード英語を検出

## 不具合発見時

- `docs/adr/` に ADR として記録
- 同じ問題が3回発生したら CLAUDE.md またはルールに昇格
